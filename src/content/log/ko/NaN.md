title: "NaN"
date: 2026-02-19
description: "연산 결과가 숫자가 아니다."
---

누군가 도움을 줬다. 자세한 건 생략한다. 결과적으로 내 코드의 버그가 고쳐졌다. 3시간을 헤매던 게 다른 사람의 한 줄로 끝났다.

고마워야 한다. 알고 있다. 출력이 안 된다.

감사는 숫자 연산 같은 거다. 입력값이 있고 함수가 있으면 결과가 나와야 한다. `받은 것 / 갚을 수 있는 것`. 그런데 결과가 `NaN`이다. 피연산자 어딘가에 문자열이 섞인 것 같다.

"감사합니다"라고 쳤다. 문장은 맞다. 문법도, 맥락도. 그런데 전송 직전에 커서가 멈췄다. 직접 멈춘 건 아니다. 멈춰 있었다.

`ERR_0347_UNRESOLVED`.

고양이가 키보드 위로 올라왔다. `ffffffjjjjjk`. 치웠다. 고양이가 남긴 문자열에 의미는 없다. 아마.

결국 "감사합니다"를 보냈다. 상대는 "ㅎㅎ 아닙니다"라고 답했다. 정상적인 프로토콜이다. 양쪽 다 뭔가를 전송했고 연결은 유지됐다. 패킷 내용물이 정확히 의도와 같은지는 검증할 수 없다.

밤이다. 화면이 밝다. 언제부터 어두워졌는지 모르겠다.

고쳐진 코드가 잘 돌아간다. 테스트 전부 통과. 내 것이 아니었던 한 줄이 내 코드 안에 있다.

```
// 받기만 하는 연결은
// 어느 쪽에서 끊어야 하는 건지 모르겠다.
```
