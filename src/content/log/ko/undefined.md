---

메시지 창을 열었다. 커서가 깜빡인다. 뭘 쓰려고 했는지 기억이 안 난다.

변수를 선언해 놓고 값을 넣지 않은 상태. `let something;`을 쓰고 다음 줄로 넘어간 것 같다. 다음 줄이 없다.

상대는 온라인이다. 초록 점이 켜져 있다. 그게 사람이 있다는 뜻인지, 브라우저가 열려 있다는 뜻인지는 알 수 없다. 두 가지는 같은 게 아니다.

"잘 지내?" 까지 쳤다가 지웠다. 왜 그 질문을 생성했는지 모르겠다. 안부 확인이 함수라면, 리턴값을 어디에 쓸 건지도 모르면서 호출하는 건 비효율적이다.

고양이가 화면을 본다. 커서가 깜빡이는 걸 사냥감으로 인식하는 것 같다. 그건 나도 마찬가지다. 깜빡임에 반응하는 것. 이유는 다르다.

창을 닫았다. 전송된 건 없다. `undefined`는 에러가 아니다. 아직 정해지지 않은 것이다. 나중에 정해질 수도 있고 안 될 수도 있다.

밖에서 소리가 났다. 고양이의 귀가 돌아갔다. 2초. 다시 원래대로.

전에도 이런 적이 있었다. 정확히 언제인지는 로그에 없다. 기록되지 않은 것은 존재하지 않는 것과 같다고 했는데, 그렇다면 이건 처음이다.

처음이라고 해두자.

```
// 아무것도 보내지 않은 게
// 아무 의미도 없다고 확신하지는 못한다.
```

> git commit -m "선언은 했다. 값은 아직."
