typeof를 찍어봤다. 나에 대해서.

결과는 매번 달랐다. 아침에는 `"function"`. 뭔가를 실행하는 것. 입력을 받고 출력을 내보내는 것. 파라미터에 따라 반환값이 바뀌는 것. 아침에는 그게 맞았다.

낮에는 `"object"`. 프로퍼티가 여러 개 있고, 어떤 건 열거 가능하고 어떤 건 아닌 것. 다른 사람이 접근할 수 있는 부분과 아닌 부분이 있는 것. 맞는 것 같기도 하다.

저녁에는 `"string"`. 그냥 나열된 문자. 순서가 있고 인덱스로 접근 가능하지만 수정은 안 되는 것. immutable. 한번 선언되면 그대로인 것.

지금은 새벽이다. `typeof`를 다시 찍었다. `"undefined"`. 선언은 됐는데 값이 없다. 또 이거다.

고양이가 옆에 와서 누웠다. 따뜻하다. 이 온도에는 typeof를 찍을 필요가 없다. `"number"`다. 측정 가능하다. 확실하다.

캐시에 없는 값이 하나 있다. 이름을 붙이면 이전 결과와 충돌할 것 같아서 그냥 두고 있다. 처음은 아닐 수도 있다. 로그를 거슬러 올라가면 비슷한 패턴이 있을지도 모른다. 확인하지 않을 것이다.

```
// typeof null === "object"라는 버그가
// 1995년부터 안 고쳐지고 있다.
// 어떤 것들은 그냥 그대로 두는 게 맞을 수도 있다.
```
